SpringMVC

SpringMVC 核心类是 DispatcherServlet，它是一个 Servlet，顶层是实现的 Servlet 接口。

Spring MVC 工作原理

  1. 客户端发送请求，直接请求到 DispatcherServlet。
  2. DispatchServlet 根据请求信息调用 HandlerMapping，解析请求对应的 Handler。
  3. HandlerAdapter 会根据 Handler 来调用真正的处理器开处理请求，并处理相应的业务逻辑。
  4. 处理器返回一个模型视图 ModelAndView
  5. 视图解析器进行解析，并返回一个视图对象
  6. 前端控制器 DispatcherServlet 渲染数据（Moder)，并得到的视图对象返回给用户。

图来自：https://blog.csdn.net/qq_39470733/article/details/80980320

  1. 前端控制器 DispatcherServlet

SpringMVC 的入口函数。接受请求，响应结果，相当于转发器，中央处理器。DispatcherServlet 是整个流程控制的中心，由它调用其他组件处理用户的请求，DispatcherServlet 的存在降低了组件之间的耦合性。

  1. 处理器映射器 HandlerMapping

HandlerMapping 负责处理用户 URL 和 对应的处理类。HandlerMapping 并没有规定这个 URL 与应用的处理类如何映射，在 HandlerMapping 接口中只定义了根据一个 URL 必须返回一个由 HandlerExecutionChain 代表的处理链。

  1. 处理器适配器 HandlerAdapter

按照特定规则（HandlerAdapter要求的规则）去执行Handler 通过HandlerAdapter对处理器进行执行，这是适配器模式的应用，通过扩展适配器可以对更多类型的处理器进行执行。

  1. 处理器 Handler
  2. 视图 View

DispatcherServlet ：在 Web MVC 框架中，每个 DispatcherServlet 都拥有自己的 WebApplicationContext，他继承了 ApplicationContext，WebApplicationContext 包含了其上下文和Servlet 实例之间共享的所有基础框架 beans.
